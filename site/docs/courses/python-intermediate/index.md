# Python Intermediate: Object-Oriented Python Through Are-Self

You have learned to read Python. Now you will learn to write it the way professionals do — with classes, inheritance, error handling, and structured patterns drawn from a production codebase.

Are-Self is a Django 6.x neurologically-inspired AI reasoning swarm engine with nine brain regions, each implemented as a Django app. In this course, you will not just study these patterns in the abstract — you will read them in the code, understand why they were built that way, and write your own.

## Philosophy

This course engages several Variables from scipraxianism:

- **Inquiry** deepens here. You are no longer asking "what does this do?" but "why was it built this way?"
- **Responsibility** expands. Your code now affects other code. Classes you write become foundations that other code depends on.
- **Fun** matters. Object-oriented programming can feel like architecture — designing structures that are elegant, efficient, and satisfying to build.
- **Humility** remains essential. The patterns in Are-Self evolved through iteration and hard-won lessons. Respect the complexity before you try to improve it.

## Prerequisites

- Completion of [Python Beginner](../python-beginner/) or equivalent experience. "Equivalent" means you are comfortable with: variables, data types, conditionals, functions, loops, lists, dictionaries, and file I/O.
- Access to the Are-Self repository
- Python 3.12+ and a text editor

## Course Modules

### Module 1: Classes and Objects
Object-oriented programming fundamentals using Are-Self's model classes — Engram, SpikeTrain, and IdentityDisc — as living examples.

### Module 2: Inheritance and Mixins
How Are-Self reuses code across every brain region through CreatedAndModifiedWithDelta, UUIDIdMixin, NameMixin, and VectorMixin. The power and discipline of multiple inheritance.

### Module 3: Error Handling
Try/except, custom exceptions, and graceful failure. How Are-Self's circuit breaker pattern handles cascading errors without crashing the system.

### Module 4: Working with APIs
The requests library, REST principles, and Django REST Framework endpoints. You will call Are-Self's API and process the responses.

### Module 5: Database Patterns
Django ORM, querysets, filtering, select_related, and prefetch_related. How Are-Self queries its PostgreSQL database efficiently.

### Module 6: Testing
Pytest, Django TestCase, assertions, and test-driven development. You will write tests for an Are-Self model and verify its behavior.

### Module 7: Signals and Events
Django signals, post_save, m2m_changed, and event-driven architecture. How Are-Self's neurotransmitter system fires signals through WebSocket connections.

### Module 8: Capstone Project
Design and implement a new model and API endpoint that extends an existing Are-Self brain region.

## Learning Outcomes

By the end of this course, you will be able to:

1. Design and implement Python classes with proper encapsulation
2. Use inheritance and mixins to share behavior across classes
3. Handle errors gracefully with try/except and custom exceptions
4. Interact with REST APIs using Python
5. Write Django ORM queries for common database operations
6. Write meaningful automated tests
7. Understand event-driven programming through Django signals
8. Extend a Django application with new models and endpoints
